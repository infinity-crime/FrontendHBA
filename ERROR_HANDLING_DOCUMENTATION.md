# Документация по обработке ошибок в портале управления кадрами

## Обзор системы обработки ошибок

Система включает несколько типов обработки ошибок:
- **Ошибки валидации** (Validation errors) - некорректные данные в формах
- **Ошибки конфликта** (Conflict errors) - дублирование данных в системе
- **Ошибки загрузки файлов** (File upload errors) - проблемы с загрузкой изображений
- **Серверные ошибки** (Server errors) - ошибки на стороне сервера
- **Ошибки сети** (Network errors) - проблемы с подключением

---

## ОШИБКА #1: Валидация входа (Пустые поля и неверный логин)

### Описание
Ошибка возникает при попытке входа в систему с пустыми полями или неверным логином.

### Действия пользователя
1. Пользователь открывает страницу входа
2. Нажимает кнопку "Войти" без заполнения полей или вводит неверный логин
3. Система показывает сообщение об ошибке

### Типы валидируемых ошибок
- Пустое поле "Логин"
- Пустое поле "Пароль"
- Пароль содержит менее 3 символов
- Логин не является "менеджер" или "админ"

### Позиция сообщения об ошибке
**Модальное диалоговое окно "ErrorModal"** - появляется в центре экрана с:
- Иконкой предупреждения (оранжевый цвет)
- Названием типа ошибки: "Ошибка валидации входа"
- Кнопкой закрытия
- Кнопкой повтора (если применимо)

**Дополнительно:** Под каждым полем ввода отображается красная линия и текст ошибки

### Компонент
**Файл:** [src/components/form/LoginForm.tsx](src/components/form/LoginForm.tsx)

**Валидатор:** [src/service/validators/formValidators.ts](src/service/validators/formValidators.ts)
```typescript
validateLoginForm(email, password)
```

### Пример валидируемых значений
| Логин | Пароль | Результат | Ошибка |
|-------|--------|-----------|--------|
| (пусто) | password | ❌ Ошибка | "Логин не может быть пустым" |
| менеджер | (пусто) | ❌ Ошибка | "Пароль не может быть пустым" |
| менеджер | "12" | ❌ Ошибка | "Пароль должен содержать минимум 3 символа" |
| неверный | password | ❌ Ошибка | "Логин должен быть \"менеджер\" или \"админ\"" |
| менеджер | любой | ✅ Успех | - |

## ОШИБКА #2: Конфликт данных при добавлении пользователя (Email уже используется)

### Описание
Ошибка возникает при попытке добавить нового пользователя, если email уже зарегистрирован в системе или логин уже существует.

### Действия пользователя
1. Администратор открывает форму добавления пользователя
2. Вводит фамилию, имя, email и другие данные
3. Нажимает кнопку "Добавить"
4. Система проверяет наличие email в БД и возвращает ошибку конфликта

### Типы валидируемых ошибок
- **409 Conflict** - Email уже используется другим пользователем
- **409 Conflict** - Логин уже используется другим пользователем
- Предварительная валидация перед отправкой:
  - Пустое имя, фамилия, отчество
  - Неверный формат email
  - Пустая дата рождения
  - Пользователь младше 18 лет
  - Неверный номер телефона
  - Не выбран отдел или должность

### Позиция сообщения об ошибке
**Модальное диалоговое окно "ErrorModal"** - появляется внутри модального окна добавления пользователя с:
- Иконкой конфликта (красный/оранжевый цвет)
- Названием типа ошибки: "Email уже используется" или "Логин уже используется"
- Подробная информация о причине конфликта
- Предложение использовать другой email

### Компонент
**Файл:** [src/components/modalWindows/AddUser.tsx](src/components/modalWindows/AddUser.tsx)

**Валидатор:** [src/service/validators/formValidators.ts](src/service/validators/formValidators.ts)
```typescript
validateUserProfile(data)
validateEmail(email)
```

**Обработчик ошибок API:** [src/service/errorHandlers/apiErrorHandler.ts](src/service/errorHandlers/apiErrorHandler.ts)
```typescript
handleConflictError(error) // Для ошибок 409
```

### Пример валидируемых значений
| Email | Результат | HTTP код | Ошибка |
|-------|-----------|----------|--------|
| ivan@example.com (уже существует) | ❌ Ошибка | 409 | "Email уже используется" |
| (пусто) | ❌ Ошибка | 400 | "Email не может быть пустым" |
| invalid.email | ❌ Ошибка | 400 | "Email должен быть в формате: example@domain.com" |
| new@example.com | ✅ Успех | 201 | - |

## ОШИБКА #3: Ошибка загрузки файла (Неверный формат или размер изображения)

### Описание
Ошибка возникает при попытке загрузить фотографию профиля неверного формата или превышающей размер допустимый размер.

### Действия пользователя
1. Пользователь открывает свой профиль
2. Нажимает кнопку "Добавить фото"
3. Выбирает файл, который:
   - Не является изображением (например: PDF, DOC)
   - Имеет размер более 5MB
   - Не поддерживаемого формата
4. Система показывает ошибку валидации

### Типы валидируемых ошибок
- Файл не выбран
- Размер файла превышает 5MB
- Неподдерживаемый формат файла (допустимые: JPEG, PNG, GIF, WebP)

### Позиция сообщения об ошибке
**Модальное диалоговое окно "ErrorModal"** - появляется на странице профиля с:
- Иконкой предупреждения (оранжевый цвет для валидации, красный для серверных ошибок)
- Названием типа ошибки: "Ошибка при загрузке файла"
- Подробная информация о требованиях к файлам:
  - Максимальный размер: 5MB
  - Допустимые форматы: JPEG, PNG, GIF, WebP
- Кнопка закрытия

### Компонент
**Файл:** [src/components/form/MainProfileForm.tsx](src/components/form/MainProfileForm.tsx)

**Валидатор:** [src/service/validators/formValidators.ts](src/service/validators/formValidators.ts)
```typescript
validateFileUpload(file, maxSizeMB = 5)
```

**Обработчик ошибок API:** [src/service/errorHandlers/apiErrorHandler.ts](src/service/errorHandlers/apiErrorHandler.ts)
```typescript
handleApiError(error) // Для ошибок загрузки на сервер
```

### Пример валидируемых значений
| Файл | Размер | Формат | Результат | Ошибка |
|------|--------|--------|-----------|--------|
| (не выбран) | - | - | ❌ Ошибка | "Файл не выбран" |
| photo.pdf | 2MB | PDF | ❌ Ошибка | "Допустимые форматы: JPEG, PNG, GIF, WebP" |
| image.jpg | 10MB | JPEG | ❌ Ошибка | "Размер файла не должен превышать 5MB" |
| photo.jpg | 2MB | JPEG | ✅ Успех | - |
| photo.png | 3MB | PNG | ✅ Успех | - |

## Компоненты системы обработки ошибок

### 1. ErrorModal (Модальное окно ошибки)

**Файл:** [src/components/modalWindows/ErrorModal.tsx](src/components/modalWindows/ErrorModal.tsx)

**Характеристики:**
- Отображает серьезные ошибки в модальном окне
- Имеет цветовое кодирование в зависимости от типа ошибки
- Поддерживает кнопку "Повторить попытку"
- Позиция: Центр экрана, поверх всех элементов

**Типы ошибок:**
- `validation` (оранжевый) - ошибки валидации данных
- `server` (красный) - ошибки на стороне сервера (500+)
- `network` (розовый) - ошибки подключения
- `permission` (фиолетовый) - доступ запрещен (403)
- `conflict` (красно-оранжевый) - конфликт данных (409)

### 2. ValidationError и FieldError (Ошибки полей)

**Файл:** [src/components/errors/ValidationError.tsx](src/components/errors/ValidationError.tsx)

**Характеристики:**
- Отображает список ошибок в контейнере
- FieldError - ошибка под отдельным полем ввода
- Позиция: Under the input field or at the top of the form
- Цвет: Красный текст с иконкой предупреждения

### 3. Валидаторы форм

**Файл:** [src/service/validators/formValidators.ts](src/service/validators/formValidators.ts)

**Функции:**
- `validateLoginForm()` - валидация входа
- `validateUserProfile()` - валидация профиля пользователя
- `validatePosition()` - валидация должности
- `validateEmail()` - валидация email
- `validateFileUpload()` - валидация загружаемого файла
- `validateNotification()` - валидация уведомлений

### 4. Обработчик ошибок API

**Файл:** [src/service/errorHandlers/apiErrorHandler.ts](src/service/errorHandlers/apiErrorHandler.ts)

**Функции:**
- `handleApiError()` - главный обработчик ошибок
- `handleConflictError()` - обработка конфликтов (409)
- `handleValidationError()` - обработка ошибок валидации (400)
- `handlePermissionError()` - обработка запретов доступа (403)
- `handleServerError()` - обработка серверных ошибок (500+)
- `handleNetworkError()` - обработка ошибок сети

## Архитектура обработки ошибок

```
Пользовательское действие
         ↓
      Форма
         ↓
   Валидатор (formValidators.ts)
         ↓
   Ошибка валидации? ──→ Показать FieldError/ValidationError
         ↓ Нет
      API запрос
         ↓
   Ответ сервера
         ↓
   обработчик ошибок (apiErrorHandler.ts)
         ↓
   Ошибка обнаружена? ──→ Показать ErrorModal
         ↓ Нет
      Успех
```
