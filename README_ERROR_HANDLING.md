# 🛡️ Система обработки ошибок портала управления кадрами

## Краткое описание

В этом документе описана полная система обработки ошибок, реализованная в приложении портала управления кадрами. Система включает валидацию на клиенте и обработку ошибок сервера с красивым UI.

---

## 📋 Быстрый старт

### Что было создано

✅ **3 компонента для отображения ошибок**
- ErrorModal - для критических ошибок
- ValidationError - список ошибок
- FieldError - ошибка под полем ввода

✅ **6 валидаторов форм**
- validateLoginForm() - проверка входа
- validateUserProfile() - профиль пользователя
- validateEmail() - email адреса
- validatePosition() - должности
- validateFileUpload() - загруженные файлы
- validateNotification() - уведомления

✅ **6 обработчиков ошибок API**
- handleApiError() - главный обработчик
- handleConflictError() - конфликты 409
- handleValidationError() - ошибки 400
- handlePermissionError() - запреты 403
- handleServerError() - ошибки 500+
- handleNetworkError() - сетевые ошибки

## 🎯 Три основные точки валидации

### 1️⃣ Форма входа (LoginForm)
**Файл:** `src/components/form/LoginForm.tsx`

**Валидируемые ошибки:**
- ✅ Пусто логин
- ✅ Пусто пароль
- ✅ Пароль < 3 символов
- ✅ Неверный логин (не "менеджер" и не "админ")

**Где видны ошибки:**
- Красная линия под полем ввода
- Красный текст ошибки
- Модальное окно ErrorModal при критических ошибках

---

### 2️⃣ Добавление пользователя (AddUser)
**Файл:** `src/components/modalWindows/AddUser.tsx`

**Валидируемые ошибки:**

**Клиент:**
- ✅ Пусто имя, фамилия, отчество
- ✅ Неверный email формат
- ✅ Неполный номер телефона
- ✅ Возраст < 18 лет
- ✅ Не выбран отдел
- ✅ Не выбрана должность

**Сервер (409 Conflict):**
- ✅ **Email уже используется**
- ✅ **Login уже используется**

**Где видны ошибки:**
- Красный текст под полем ввода
- Модальное окно ErrorModal (красный) для конфликтов

---

### 3️⃣ Загрузка фотографии (MainProfileForm)
**Файл:** `src/components/form/MainProfileForm.tsx`

**Валидируемые ошибки:**
- ✅ Неподдерживаемый формат (не JPEG, PNG, GIF, WebP)
- ✅ **Файл слишком большой (> 5MB)**
- ✅ Файл не выбран

**Где видны ошибки:**
- Модальное окно ErrorModal (оранжевый) с подробной информацией

3. Увидите: ErrorModal (оранжевый) "Ошибка валидации входа"

## 🎨 Типы ошибок и их цвета

| Тип | Цвет | Примеры | HTTP |
|-----|------|---------|------|
| `validation` | 🟠 Оранжевый | Пустые поля, неверный формат | 400 |
| `conflict` | 🔴 Красный | Email существует, логин дублируется | 409 |
| `server` | 🔴 Красный | Ошибка базы данных, Exception | 500+ |
| `permission` | 🟣 Фиолетовый | Менеджер пытается в админ-панель | 403 |
| `network` | 🟡 Розовый | Сервер недоступен, timeout | - |

## 💻 Примеры кода

### Использование валидации в форме
```tsx
import { validateLoginForm } from '../../service/validators/formValidators';
import { ErrorModal } from '../modalWindows/ErrorModal';

export const MyForm = () => {
  const handleSubmit = () => {
    // Валидировать данные
    const validation = validateLoginForm(email, password);
    
    if (!validation.isValid) {
      // Показать ошибки
      setErrorModal({
        isOpen: true,
        title: 'Ошибка валидации',
        message: 'Проверьте введенные данные',
        errorType: 'validation',
      });
      return;
    }
    
    // Отправить форму...
  };
};
```

### Обработка ошибок API
```tsx
import { handleApiError } from '../../service/errorHandlers/apiErrorHandler';

try {
  const response = await apiCall();
} catch (error) {
  const apiError = handleApiError(error);
  
  setErrorModal({
    isOpen: true,
    title: apiError.title,
    message: apiError.message,
    details: apiError.details,
    errorType: apiError.type,
  });
}
```
