## 🔍 КРАТКОЕ ОПИСАНИЕ КАЖДОЙ ОШИБКИ

### ✅ ОШИБКА 1: Валидация входа
- **Где:** `src/components/form/LoginForm.tsx`
- **Валидатор:** `validateLoginForm()`
- **Ошибки:** Пусто логин, неверный логин, пусто пароль, короткий пароль
- **Отображение:** ErrorModal + FieldError
- **Где видна:** На странице входа, в центре экрана

### ✅ ОШИБКА 2: Email конфликт (409)
- **Где:** `src/components/modalWindows/AddUser.tsx`
- **Валидатор:** `validateUserProfile()` + API обработчик
- **Ошибка:** Email уже существует (HTTP 409 Conflict)
- **Обработчик:** `handleConflictError()`
- **Отображение:** ErrorModal (красный, 409)
- **Где видна:** В модальном окне добавления пользователя

### ✅ ОШИБКА 3: Файл слишком большой
- **Где:** `src/components/form/MainProfileForm.tsx`
- **Валидатор:** `validateFileUpload()`
- **Ошибка:** Размер > 5MB или неверный формат
- **Отображение:** ErrorModal (оранжевый)
- **Где видна:** На странице профиля, в центре экрана

## 👨‍💻 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Валидаторы (в `src/service/validators/formValidators.ts`)
```typescript
validateLoginForm(email, password)        // Проверка входа
validateUserProfile(data)                 // Профиль пользователя
validateEmail(email)                      // Email адрес
validatePosition(data)                    // Должность
validateFileUpload(file, maxSizeMB)       // Файлы
validateNotification(data)                // Уведомления
```

### Обработчики ошибок (в `src/service/errorHandlers/apiErrorHandler.ts`)
```typescript
handleApiError(error)                     // Маршрутизация по HTTP статусу
handleConflictError(error)                // 409 Conflict (Email существует)
handleValidationError(error)              // 400 Bad Request
handlePermissionError(error)              // 403 Forbidden
handleServerError(error)                  // 500+ Server Error
handleNetworkError(error)                 // Сетевые ошибки
```

### Типы ошибок
```typescript
type ErrorType = 'validation' | 'server' | 'network' | 'permission' | 'conflict'

// validation  🟠 Оранжевый  - Ошибки валидации
// server      🔴 Красный    - Ошибки сервера
// conflict    🔴 Красный    - Конфликты 409
// permission  🟣 Фиолетовый - Доступ 403
// network     🟡 Розовый    - Сетевые ошибки
```
