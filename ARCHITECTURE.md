## Три основные точки обработки ошибок

### 1️⃣ LOGIN FORM → Валидация входа

```
Пользователь
    │
    ▼
Вводит данные
    │
    ▼
Нажимает "Войти"
    │
    ▼
validateLoginForm(email, password)
    │
    ├─ Проверка: email пуст?
    │   └─ ДА → errors.email = ["Логин не может быть пустым"]
    │   └─ НЕТ → Проверка следующего
    │
    ├─ Проверка: это "менеджер" или "админ"?
    │   └─ НЕТ → errors.email = ["Логин должен быть 'менеджер' или 'админ'"]
    │   └─ ДА → Проверка следующего
    │
    ├─ Проверка: password пуст?
    │   └─ ДА → errors.password = ["Пароль не может быть пустым"]
    │   └─ НЕТ → Проверка следующего
    │
    ├─ Проверка: password.length < 3?
    │   └─ ДА → errors.password = ["Пароль должен содержать минимум 3 символа"]
    │   └─ НЕТ → Валидация успешна
    │
    ▼
Ошибки найдены?
    │
    ├─ ДА → Показать ErrorModal "Ошибка валидации входа"
    │       + FieldError под каждым полем
    │
    └─ НЕТ → Выполнить вход
            └─ API запрос
            └─ Редирект на /catalog
```

---

### 2️⃣ ADD USER MODAL → Конфликт данных (Email уже существует)

```
Пользователь
    │
    ▼
Заполняет форму добавления пользователя
    │
    ▼
Нажимает "Добавить"
    │
    ▼
validate() - клиентская валидация
    │
    ├─ Проверка: все обязательные поля пусты?
    │   └─ ДА → Показать красный текст под полем → Вернуться
    │   └─ НЕТ → Продолжить
    │
    └─ Проверка: email правильный формат?
        └─ НЕТ → Показать красный текст → Вернуться
        └─ ДА → Продолжить
    
    ▼
API запрос: addUser(userData)
    │
    └─ HTTP 409 CONFLICT
        └─ Response: "Email already exists"
    
    ▼
handleApiError(error)
    │
    ├─ Проверка: error.response.status === 409?
    │   └─ ДА → handleConflictError()
    │
    ▼
handleConflictError()
    │
    └─ Возвращает: {
        type: 'conflict',
        title: 'Email уже используется',
        message: 'Пользователь с таким email...',
        details: 'Пожалуйста, используйте другой email'
    }
    
    ▼
setErrorModal({ isOpen: true, ... })
    │
    ▼
Показать ErrorModal (красный, 409 Conflict)
```

### 3️⃣ MAIN PROFILE FORM → Ошибка загрузки файла

```
Пользователь
    │
    ▼
Нажимает "Добавить фото"
    │
    ▼
Выбирает файл
    │
    ▼
handleFileSelect(file)
    │
    ▼
validateFileUpload(file, maxSizeMB=5)
    │
    ├─ Проверка: file === null?
    │   └─ ДА → errors.file = ["Файл не выбран"]
    │
    ├─ Проверка: file.size > 5MB?
    │   └─ ДА → errors.file = ["Размер файла не должен превышать 5MB"]
    │
    ├─ Проверка: расширение поддерживается?
    │   (JPEG, PNG, GIF, WebP)
    │   └─ НЕТ → errors.file = ["Допустимые форматы: JPEG, PNG, GIF, WebP"]
    │
    └─ ДА → Файл правильный
    
    ▼
Ошибки?
    │
    ├─ ДА → Показать ErrorModal "Ошибка при загрузке файла" (оранжевый)
    │       + Детали об ограничениях
    │
    └─ НЕТ → Отправить файл на сервер
            │
            ▼
            API: AddPhotoProfile(formData)
            │
            ├─ Успех → Toast "Фото добавлено!"
            │
            └─ Ошибка → handleApiError()
                └─ Показать ErrorModal (красный/розовый)
```

## Типы ошибок и их обработка

```
HTTP СТАТУС КОД
    │
    ├─── 400 Bad Request
    │    └─ handleValidationError()
    │       └─ type: 'validation'
    │       └─ color: 🟠 Orange
    │       └─ title: "Ошибка валидации данных"
    │       └─ errorModal: YES
    │
    ├─── 403 Forbidden
    │    └─ handlePermissionError()
    │       └─ type: 'permission'
    │       └─ color: 🟣 Purple
    │       └─ title: "Доступ запрещен"
    │       └─ errorModal: YES
    │
    ├─── 409 Conflict
    │    └─ handleConflictError()
    │       └─ type: 'conflict'
    │       └─ color: 🔴 Red
    │       └─ title: "Email уже используется"
    │       └─ errorModal: YES
    │       └─ Примеры: Email exists, Login exists, Duplicate data
    │
    ├─── 500+ Server Error
    │    └─ handleServerError()
    │       └─ type: 'server'
    │       └─ color: 🔴 Red
    │       └─ title: "Ошибка сервера"
    │       └─ errorModal: YES
    │
    └─── Network Error
         └─ handleNetworkError()
            └─ type: 'network'
            └─ color: 🟡 Pink
            └─ title: "Ошибка подключения"
            └─ errorModal: YES
```

## Мониторинг и отладка

```
Для отладки ошибок используйте:

Console → Network:
├─ Проверьте HTTP статус код
├─ Посмотрите Response body
└─ Проверьте Headers

Console → Application:
├─ localStorage - сохраненные токены
└─ Session Storage

Console → Logs:
└─ console.log('API Error:', error)
   console.error('Validation:', validation)

ErrorModal показывает:
├─ type ошибки
├─ title
├─ message
├─ details (если есть)
└─ statusCode (для API errors)
```
